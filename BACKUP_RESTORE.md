# BACKUP_RESTORE

## Что сохраняется
- SQLite база целиком (`smeta.sqlite`), включая:
  - документы и snapshot-цепочку,
  - платежи и аллокации,
  - numbering,
  - шаблоны документов,
  - каталоги и расчётные данные.

## Backup
1. Откройте **Settings**.
2. Нажмите `Backup базы`.
3. Выберите путь. Имя по умолчанию: `smeta-backup-YYYYMMDD-HHMMSS.sqlite`.

## Restore
1. Откройте **Settings**.
2. Нажмите `Restore базы`.
3. Подтвердите замену рабочей базы.
4. Приложение валидирует backup по обязательным таблицам перед восстановлением.

## Риски
- Restore полностью заменяет текущую рабочую БД.
- Перед restore всегда делайте дополнительный backup текущей базы.
